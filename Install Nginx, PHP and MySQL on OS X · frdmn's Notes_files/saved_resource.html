                                        • Scott Prock • 2 years ago

                                          I was getting 502 Bad Gateway when trying to access a virtual host. Tracked it down to Unix sockets. The error I got was ... unix:/var/run/php5-fpm.sock failed ... There's a discussion further down with options to try and I found that changing my configs to this ...

                                          Change the Nginx vhost from:
                                          fastcgi_pass unix:/var/run/php5-fpm.sock;

                                          To
                                          fastcgi_pass 127.0.0.1:9000;

                                          which matches the listen = in php-fpm.conf

                                          When I tried to use the unix socket by changing the php-fpm.con to match my Nginx config then restarting php5-fpm would fail.

                                          Hope this helps someone else ... Scott

                                                                                                                        index index.php index.html;
                                                                                                                        location ~ ^/phpmyadmin/(.*\.php)$ {
                                                                                                                            root /usr/local/share/;
                                                                                                                            include conf.d/php-fpm;
                                                                                                                        }
                                                                                                                    }
